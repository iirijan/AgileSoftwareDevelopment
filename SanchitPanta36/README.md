
## Step 1: Fork the Repository
- Go to the original repository
- Click on **Fork** (top-right corner)
- This creates a copy in your GitHub account

---

## Step 2: Open Your Fork
- Go to your GitHub profile
- Open the forked repository

---

## Step 3: Create a New Folder
GitHub does not allow empty folders, so we create a folder using a file.

- Click **Add file → Create new file**
- In the filename box, type:


YourNameRoll/README.md


- Add any simple text inside:
My Lab Work Folder

- Click **Commit changes**

---

## Step 4: Upload PDF File
- Open your created folder
- Click **Add file → Upload files**
- Drag and drop your PDF file
- Example:

Lab1_Agile.pdf


- Make sure the path is:

YourNameRoll/Lab1_Agile.pdf


- Click **Commit changes**

---

## Step 5: Submit Pull Request
- Click **Contribute → Open pull request**
- Submit the pull request to the original repository
- The owner will review and merge it

---

## Important Notes
- GitHub folders are created automatically from file paths
- You cannot create empty folders
- Always work inside your fork if you don’t have write access

---

## Done!
Now your file is successfully submitted to the repository.
